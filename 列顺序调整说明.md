## 📝 调整原因

将"取消订阅率"放在"投诉率"前面，使得列的顺序更符合逻辑：

**用户行为指标组**（按照用户操作流程）：
1. **打开率** - 收到邮件后打开
2. **点击率** - 打开后点击链接
3. **取消订阅率** - 不感兴趣，选择退订
4. **投诉率** - 极度不满，标记为垃圾邮件

**邮件系统指标组**：
1. **送达率** - 系统层面的送达成功率
2. **弹回率** - 系统层面的弹回率

## ✨ 用户体验优化

新的列顺序更加直观：

### 之前的顺序
```
打开率 → 点击率 → 投诉率 → 送达率 → 弹回率 → 取消订阅率
         ↓
    (用户行为混乱)
```

### 现在的顺序
```
打开率 → 点击率 → 取消订阅率 → 投诉率 | 送达率 → 弹回率
         ↓                             ↓
    (用户行为清晰)              (系统指标)
```

---

## 📊 指标分组逻辑

### 第一组：用户参与度（积极行为）
- **打开率** - 用户对邮件感兴趣
- **点击率** - 用户对内容感兴趣

### 第二组：用户反馈（消极行为）
- **取消订阅率** - 温和的拒绝（正常的退订流程）
- **投诉率** - 强烈的拒绝（标记为垃圾邮件）

### 第三组：技术指标
- **送达率** - 邮件能否成功送达
- **弹回率** - 邮件是否被退回
